	Резюме, ТЗ:

1.	формат имени одной самой минимальной УП, каждая деталь может иметь таких файликов порядка 20-30шт
	cAA-sB-nCCC-tDD.NC, где
	AA - СК и принадлежность к детали, чет и нечет, соседние номера, например 35 и36
	B - номер серии проходов для каждой СК c 1. может быть 1, 2, 3 - вряд ли больше
	ССС - номер прохода в серии, обычно типа 010, 020 и тд для возможности сделать вставку. 
	DD - номер инструмента. нужно отслеживать ситуации когда следующий номер прохода с тем же инструментом - не переходить к следующему файлу, а добавлять этот проход в последовательность
	NC - расширение файла
	
например одна одиночная УП: 35-1-010-03.NC или с35-s1-n010-t03.NC
	
2.	файлы по каждой детали вручную можно легко разделить на несколько папок или тягать их из кучи. например раскидать по папкам одну деталь - УП начинающиеся  
35-1, 35-2, 36-1, 36-2, 36-3. допустим файлы раскиданы вручную в папки соответственно 35-1 и тд

соответственно сначала для обработки выбираются файлы деталей с нечетным AA и B=1, т.е. из папок 31-1,33-1,35-1 или начинающиеся с этих цифр если все в куче. следующая конвертация файлы 31-2,33-2,35-2


3. при запуске конвертера порядок перебора AA желательно уметь задавать не по порядку, а как угодно, чтобы фреза меньше металась при переходе от детали к детали. т.е. конвертер должен читать еще вручную созданный файлик Order.txt типа:
	n1 = 31
	n2 = 33
	n3 = 45	
	.
	.
4. пока предлагается не склеивать, а складывать результаты конвертации в выходную папку. таких папок получится = 
максимальное количество серий с кажой стороны. столько же как и входных, как ориентир от 4 до 6

формат названия по сути только переименованных файла в каждой папке
e(o)E-bFF-sGG-cHH-tII, где
E - номер пакета (который может быть выполнен не трогая деталь). первая e или o (even/odd) и там по порядку e1,e2,e3,o1,o2 
FF - порядок выполнения прохода одним инструментом (bunch)
GG - порядок выполнения внутри одного прохода (sequence)
HH - номер СК (только для информации) (coordinate)
II - номер инструмента Т (только для информации) (tool)

т.е. вид типа E1-05-22-35-01.NC или e1-b05-s22-c35-t01.NC
возможно по букве искать проще.
