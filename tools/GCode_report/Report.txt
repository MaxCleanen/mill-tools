
- WLMill может выполнять внешние exe и cmd

если есть cmd - склеить файлы можно средствами ОС

- нет возможности прочитать файл чтобы чтото в нем найти. средствами ОС можно сделать какие то поиски и отчеты - но их тоже по нормальному не считать, так как формат отчетов не совпадает с форматом который может прочитать WLM.

считать из файла данные WLM может если файл организован типа:
[Program]
file=C:/Users/VT/Desktop/Gcodes/Source_NC/01-MAC2T4.NC
iLastElement=0
iLast_WorkElement=0
autoLoad=false

в УП есть строки с определенным номером типа:
 - O1001 - отсюда можно взять номер работы. сейчас приходится загружать УП и тогда можно по номеру строки его взять - это не очень хорошо.
 - (01-MAC2T4  0:00:10) - отсюда можно взять время выполнения этой УП
 - T4 M06 отсюда можно взять номер задействованного инструмента Т - это нужно чтобы сверить с тем что в названии файла или для переименования файла
 - G00 G90 G154 P1 - отсюда можно взять номер СК, G154 P1 (если используем только расширенные) для сверки с именем файла
 
прим - номер строки может быть изменен и хорошо бы его брать из ини файла config2.ini
  
может быть возникнет в перспективе необходимость найти в УП чтото по ключевому слову. но это скорее для уже склеенных УП. 

то есть была бы полезна утилита формирующая из заданного файла УП файл типа
[имя УП]
строка N=
строка M=
.
.

тогда уже этот файл WLM могла бы считать, интерпретировать и потом стереть

если в задании не имя а шаблон то можно в один отчет данные всех файлов
т.е.
